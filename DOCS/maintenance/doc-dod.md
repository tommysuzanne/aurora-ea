# Definition of Done (DoD) — documentation

À appliquer à **chaque** changement significatif dans `DOCS/`.

## DoD

- Les liens internes `DOCS/` sont valides (index → feuilles, feuilles → index).
- Les chemins cités sont relatifs au repo (ex: `MQL5/Experts/Aurora.mq5`).
- Les nouveaux faits non présents dans les sources sont remplacés par un `TODO(verify)` avec “comment obtenir”.
- Les divergences doc vs code sont annotées `TODO(reconcile)` avec une méthode.
- Les index concernés (`*/index.md`) sont mis à jour.
- Les docs critiques ont une section `Last verified`.
